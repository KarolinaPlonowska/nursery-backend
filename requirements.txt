# =============================
# Wymagania systemowe – System zarządzania żłobkiem
# =============================
#
# 1. Wymagania funkcjonalne
# -------------------------
# 1.1. Uwierzytelnianie i autoryzacja
# - System umożliwia rejestrację i logowanie użytkowników (rodzic, opiekun, administrator).
# - Po zalogowaniu użytkownik otrzymuje token JWT.
# - Dostęp do funkcji systemu jest kontrolowany przez role (RBAC).
#
# 1.2. Zarządzanie użytkownikami
# - Administrator może przeglądać, dodawać i edytować użytkowników.
# - Administrator może przypisywać role i grupy użytkownikom.
#
# 1.3. Zarządzanie dziećmi
# - Rodzic może dodawać i przeglądać swoje dzieci.
# - Opiekun widzi dzieci przypisane do swojej grupy.
# - Administrator widzi wszystkie dzieci.
#
# 1.4. Zarządzanie grupami
# - Administrator może tworzyć i edytować grupy.
# - Administrator może przypisywać dzieci i opiekunów do grup.
#
# 1.5. Obsługa obecności (opcjonalnie)
# - Opiekun może rejestrować obecność dzieci w grupie.
#
# 1.6. Bezpieczeństwo
# - Hasła są hashowane (bcrypt).
# - Endpoint logowania chroniony rate-limiterem.
# - Dane użytkowników są walidowane po stronie frontend i backend.
# - Dostęp do danych ograniczony do uprawnień użytkownika.
#
# 2. Wymagania niefunkcjonalne
# ----------------------------
# 2.1. Technologia
# - Backend: NestJS, TypeORM, PostgreSQL, bcrypt, winston (logowanie)
# - Frontend: React, Ant Design, axios, react-router-dom
# - Komunikacja: REST API, JWT
#
# 2.2. Wydajność i skalowalność
# - System obsługuje jednoczesną pracę wielu użytkowników.
# - Backend i frontend mogą być uruchamiane w kontenerach Docker.
#
# 2.3. Bezpieczeństwo
# - Dane przesyłane przez HTTPS.
# - Ograniczenie CORS do zaufanych domen.
# - Logowanie zdarzeń i błędów.
# - Regularne backupy bazy danych.
#
# 2.4. Utrzymanie i rozwój
# - Kod podzielony na moduły (users, children, groups, auth, logger).
# - Testy jednostkowe i integracyjne.
# - Dokumentacja API i kodu.
#
# 3. Wymagania środowiskowe
# -------------------------
# - Node.js (zalecana najnowsza LTS)
# - PostgreSQL 13+
# - Docker (opcjonalnie)
# - System operacyjny: Linux/Windows
